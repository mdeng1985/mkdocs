    42的makefile学习及cmake改写¶

    makefile #cmake¶

    *FLAGS¶

    CFLAGS¶

    CFLAGS = -Wall -Wshadow -Wno-deprecated -g $(ANSIFLAGS) $(GLINC) $(CINC) -I $(INC) -I $(KITINC) -I $(KITSRC) $(GMSECINC) -O0 $(ARCHFLAG) $(GUIFLAG) $(SHADERFLAG) $(CFDFLAG) $(FFTBFLAG) $(GSFCFLAG) $(GMSECFLAG) $(STANDALONEFLAG) $(NOS3FSWFLAG) $(GLFWFLAG)

    其中：

    • ANSIFLAGS有两种情况：
    *#ANSIFLAGS = -Wstrict-prototypes -pedantic -ansi -Werror*

ANSIFLAGS =

    • CINC =

    • INC = $(PROJDIR)Include/

    • KITINC = $(KITDIR)Include/其中：KITDIR = $(PROJDIR)Kit/

    • KITSRC = $(KITDIR)Source/

    • GMSECINC的值取决于GMSECFLAG，如果GMSECFLAG为空，则GMSECINC也为空。如果*GMSECFLAG = -D _ENABLE_GMSEC_*，则GMSECINC = -I $(GMSECDIR)include/

    • 如果GUIFLAG为空，这时候不需要显示界面，则：

    GUIOBJ =
GLINC =
LIBS = -ldl -lm
LFLAGS =
ARCHFLAG =

    否则：

    GUIOBJ = $(OBJ)42GlutGui.o $(OBJ)glkit.o
#GLINC = -I /usr/include/
GLINC = -I $(KITDIR)/include/GL/
LIBS = -lglut -lGLU -lGL -ldl -lm
LFLAGS = -L $(KITDIR)/GL/lib/
ARCHFLAG =
    • SHADERFLAG = -D _USE_SHADERS_ 因此gcc在CFLAGS调用的时候为gcc -D _USE_SHADERS_

    含义就是gcc定义了宏_USE_SHADERS_，该宏的默认值为1. see[[gcc学习#gcc -D选项]] 。推断程序里用到了_USE_SHADERS_这个宏

    • makefile CFDFLAG = #CFDFLAG = -D _ENABLE_CFD_SLOSH_

    • makefile FFTBFLAG = #FFTBFLAG = -D _ENABLE_FFTB_CODE_

    • ```makefile #GSFCFLAG =

    GSFCFLAG = -D USE_GSFC_WATERMARK ```

    • makefile GMSECFLAG =

    • ```makefile STANDALONEFLAG =

    #STANDALONEFLAG = -D AC_STANDALONE ```

    • makefile NOS3FSWFLAG = #NOS3FSWFLAG = -D _ENABLE_NOS3_FSW_

    • makefile #GLFWFLAG = GLFWFLAG = -D _USE_GLFW_

    LFLAGS¶

    取决于GUIFLAG，如果GUIFLAG = -D _USE_GUI_, 则LFLAGS = -L $(KITDIR)/GL/lib/

    否则为空。

    生成42可执行文件¶

    42 : $(42OBJ) $(GUIOBJ) $(SIMIPCOBJ) $(FFTBOBJ) $(SLOSHOBJ) $(KITOBJ) $(ACOBJ) $(GMSECOBJ)
    $(CC) $(LFLAGS) $(GMSECBIN) -o $(EXENAME) $(42OBJ) $(GUIOBJ) $(FFTBOBJ) $(SLOSHOBJ) $(KITOBJ) $(ACOBJ) $(GMSECOBJ) $(SIMIPCOBJ) $(LIBS) $(GMSECLIB)

    • 42OBJ变量由一系列.o文件组成，见下文

    • GUIOBJ取决于GUIFLAG, 没有界面则为空

    • SIMIPCOBJ取决于GMSECFLAG，如果为空的话，则：

    SIMIPCOBJ = $(OBJ)SimWriteToFile.o $(OBJ)SimWriteToSocket.o \
      $(OBJ)SimReadFromFile.o $(OBJ)SimReadFromSocket.o

    由4个.o文件组成

    • FFTBOBJ取决于FFTBFLAG，后者为空的话，则前者也为空。默认为空。暂时不管

    • SLOSHOBJ取决于CFDFLAG，默认为空

    • KITOBJ

    KITOBJ = $(OBJ)dcmkit.o $(OBJ)envkit.o $(OBJ)fswkit.o $(OBJ)geomkit.o \
$(OBJ)iokit.o $(OBJ)mathkit.o $(OBJ)nrlmsise00kit.o $(OBJ)msis86kit.o \
$(OBJ)orbkit.o $(OBJ)radbeltkit.o $(OBJ)sigkit.o $(OBJ)sphkit.o $(OBJ)timekit.o

    • ACOBJ取决于STANDALONEFLAG，STANDALONEFLAG为空的话（默认），则ACOBJ = $(OBJ)AcApp.o，否则ACOBJ为空。

    • GMSECOBJ取决于GMSECFLAG，默认为空

    • GMSECBIN取决于GMSECFLAG，默认为空

    • EXENAME = 42

    • GMSECLIB取决于GMSECFLAG，默认为空

    主要思路和步骤¶

    makefile cmakeLists
    设置变量 KITDIR = $(PROJDIR)Kit/ set(KITDIR ${PROJECT_SOURCE_DIR}/Kit)
    设置包含路径 先将-I $(INC)赋值给CFLAGS；然后在生成.o文件的时候调用CFLAGS include_directories( ${INC} )
    添加宏定义 SHADERFLAG = -D_USE_SHADERS_然后gcc ${SHADERFLAG}的时候相当于 gcc编译的时候定义了宏_USE_SHADERS_(格式gcc D宏名) | add_definitions(-D_USE_SHADERS_ )这里USE_SHADERS`为宏
    添加库 LIBS = -ldl -lm,然后gcc的时候调用这些库 target_link_libraries(42mainexe -lm -ldl)
    添加源文件 file(GLOB SOURCES42 "${SRC}/*.c" ）
    去掉不必要的文件 list(REMOVE_ITEM SOURCES42 "${PROJECT_SOURCE_DIR}/Source/IPC/AppReadFromCmd.c"）

    改写的时候几点注意事项：

    1. set(SRC ${PROJECT_SOURCE_DIR}/Source)后面使用SRC目录下的文件的时候，需要加/
    2. 程序里有多个main程序，需要仅保留一个。后面考虑可以添加多个执行文件
    3. 有些文件取决于特定变量GMSECFLAG的值，因此需要根据GMSECFLAG的值去除一些不编译的文件；其他的还有GUIFLAG、
