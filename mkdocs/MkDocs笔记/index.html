    MkDocs笔记¶

    安装与基本操作¶

    • 在wsl bash下安装和创建一个my-project 

      pip3 install mkdocs
mkdocs -v
mkdocs new my-project

    • 本地预览 

      cd my-project
mkdocs serve
      打开http://127.0.0.1:8000/, 就可以在本地预览了

    • 如果需要生成html,则mkdocs build就可以了

    • 如果需要发布到github上, 则需要mkdocs gh-deploy, 详见 后文[[MkDocs笔记#^a2d1bd|我的mkdocs实践及发布]]

    MkDocs Awesome Pages Plugin¶

    MkDocs Awesome Pages Plugin

    安装¶

    pip3 install mkdocs-awesome-pages-plugin

    典型使用¶

    在mkdocs.yml里设置如下:

    # Plugins
plugins:
  - search
  - awesome-pages:
      filename: .pages.yml        # 指定配置文件名，默认文件名是 .pages
      collapse_single_pages: false # 当目录只有一个文件时是否收缩目录
      strict: false               # 当 arrange 配置的文件不存在是是否报错
nav:
      - 首页: index.md
      - ...
    就会自动根据子目录, 生成文档.

    测试表明: - 不写.pages.yml也是没有关系的 -

    注意事项¶

    当mkdocs.yml里有nav和pages入口的话, 该插件不起作用. 解决方式有两种: 1. 删除这些入口 2. 添加...入口

    调用外部文件夹中的源文件¶

    安装markdown-include 

    pip3 install markdown-include
    然后在yml里设置: 
    markdown_extensions:
  - markdown_include.include:
      base_path: docs

    比如要有如下目录

    .
├── mkdocs/docs 
├── v1/c.md
└── v2

    需要将v1目录下的c.md加到网页里, 则只用在docs下建立一个c.md, 其中 c.md的内容为: `` 就可以了.

    上面给出的是md文件相对于docs的路径

    设置密码保护¶

    mkdocs-encryptcontent-plugin

    安装pip3 install mkdocs-encryptcontent-plugin

    在mkdocs.yml里设置全局密码: 

    plugins:
    - encryptcontent:
        global_password: 'your_password'

    ^f27725

    • 下载 abcjs-plugin-min.js, 并将该文件放入到docs/javascripts目录下

      extra_javascript:

    发布到github的流程¶

    ^a2d1bd

    1. 首先在github创建一个仓库名为mkdocs, 选中Add a README file,
    2. wsl下在knowldegBase目录下 git clone https://github.com/我的账号/mkdocs.git
    3. bash下cd mkdocs
    4. 在mkdocs目录下创建一个docs用于存放发布用的md文件; 将docs目录下的所有文件都复制到docs中
    5. 在knowldegBase\mkdocs目录下创建一个mkdocs.yml
      1. 设置源文件目录docs_dir: 'docs'
    6. mkdocs gh-deploy 发布网页到github上. 打开https://我的账号.github.io/mkdocs/即可

    TIPS: 1. 我都是在wsl下操作的 2. docs_dir目录下的所有md文件都会被转换, 即使没有在yml里没有给出链接. 3. 可以利用pip3 install mkdocs-monorepo-plugin插件实现多个yml的嵌套 (Notes: 非最新状态) 

        nav:
      - 首页: index.md
      - 兴趣:
        - Installation: getting-started.md
      - 工作:
        - Changing the colors: setup/changing-the-colors.md
      - 办公:
        - Abbreviations: reference/abbreviations.md
        - mkdocs: mkdocs语法.md
      - Insiders:
        - Sponsorship: insiders/index.md
        - v1: '!include ./v1/mkdocs.yml'
        - v2: '!include ./v2/mkdocs.yml'
    实际的目录如下: 
    .
├── docs
├── material-mkdocs-github.txt
├── mkdocs.yml
├── site
├── v1
└── v2
    其中v1和v2目录下含有yml文件和doc文件夹.

    发布到gitee的流程¶

    假设已经可以发布到GitHub了, 这里需要同步发布到gitee, 步骤如下:

    1. gitee上建个仓库mkdocs
    2. bash下进行添加gitee仓库

    git remote -v
git remote add gitee https://gitee.com/mdeng1985/mkdocs.git
    4. 编译网页mkdocs build 5. 更新site文件夹内容

    ```bash
 git add site  
git commit -m up    
 ```
    1. 推送到gitee: git push -u gitee
    2. gitee的页面操作:
      1. mkdocs仓库页面-服务-gitee pages, 设置部署分支(我的分支是main)和部署目录即可. 部署目录填site目录.
      2. 打开我的网页即可:http://mdeng1985.gitee.io/mkdocs/

    总结: 1. github分支是gp-pages 2. gitee分支是main分支.

    相关配置¶

    • 安装MkDocs Awesome Pages Plugin, 该插件的作用就是不用在mkdocs.yml里对源文件进行组织. yml的和该插件相关的内容如下: 
      plugins:
      - awesome-pages:
          filename: .pages.yml        # 指定配置文件名，默认文件名是 .pages
          collapse_single_pages: false # 当目录只有一个文件时是否收缩目录
          strict: false               # 当 arrange 配置的文件不存在是是否报错
nav:
      - 首页: index.md
      - ...
      注意: .pages.yml可写可不写.
    • 安装markdown-include插件, 该插件的作用就是在docs目录下可以调用外部的markdown文件, 具体用法见[[MkDocs笔记#^c7ac78]]
    • 安装abcjs-plugin-min.js, 该js的作用是正确显示五线谱, 详细用法[[MkDocs笔记#^f27725]]
    • yml设置公式支持, 见[[MkDocs笔记#^ac27f3]]
