    MkDocs笔记¶

    安装与基本操作¶

    • 在wsl bash下安装和创建一个my-project 

      pip3 install mkdocs
mkdocs -v
mkdocs new my-project

    • 本地预览 

      cd my-project
mkdocs serve
      打开http://127.0.0.1:8000/, 就可以在本地预览了

    • 如果需要生成html,则mkdocs build就可以了

    • 如果需要发布到github上, 则需要mkdocs gh-deploy, 详见 后文[[MkDocs笔记#^a2d1bd|我的mkdocs实践及发布]]

    主题安装与使用¶

    以rtd-dropdown为例进行说明

    pip3 install mkdocs-rtd-dropdown

    然后在yaml文件中添加theme: rtd-dropdown就可以了

    MkDocs Awesome Pages Plugin¶

    MkDocs Awesome Pages Plugin

    安装¶

    pip3 install mkdocs-awesome-pages-plugin

    典型使用¶

    在mkdocs.yml里设置如下:

    # Plugins
plugins:
  - search
  - awesome-pages:
      filename: .pages.yml        # 指定配置文件名，默认文件名是 .pages
      collapse_single_pages: false # 当目录只有一个文件时是否收缩目录
      strict: false               # 当 arrange 配置的文件不存在是是否报错
nav:
      - 首页: index.md
      - ...
    就会自动根据子目录, 生成文档.

    测试表明: - 不写.pages.yml也是没有关系的 -

    注意事项¶

    当mkdocs.yml里有nav和pages入口的话, 该插件不起作用. 解决方式有两种: 1. 删除这些入口 2. 添加...入口

    调用外部文件夹中的源文件¶

    安装markdown-include 

    pip3 install markdown-include
    然后在yml里设置: 
    markdown_extensions:
  - markdown_include.include:
      base_path: docs

    比如要有如下目录

    .
├── mkdocs/docs 
├── v1/c.md
└── v2

    需要将v1目录下的c.md加到网页里, 则只用在docs下建立一个c.md, 其中 c.md的内容为: `` 就可以了.

    上面给出的是md文件相对于docs的路径

    js配置¶

    支持显示五线谱¶

    • 下载 abcjs-plugin-min.js, 并将该文件放入到docs/javascripts目录下
    • 然后mkdocs.yml里添加: 
      extra_javascript:
    - 'javascripts/abcjs-plugin-min.js'
    • 就可以渲染md中的五线谱了,示例如下: 
          ```abc
    X: 1
    T: Cooley's
    M: 4/4
    L: 1/8
    K: Emins
    |:D2|"Em"EB{c}BA B2 EB|~B2 AB dBAG|\
        "D"FDAD BDAD|FDAD dAFD| DD DD
        "D"afe^c dBAF|"Em"DEFD E2:| 
        "D"afe^c dBAF|"Em"DEFD E2:|
    ```

    我的mkdocs实践及发布¶

    ^a2d1bd

    1. 首先在github创建一个仓库名为mkdocs, 选中Add a README file,
    2. wsl下在knowldegBase目录下 git clone https://github.com/我的账号/mkdocs.git
    3. bash下cd mkdocs
    4. 在mkdocs目录下创建一个docs用于存放发布用的md文件; 将docs目录下的所有文件都复制到docs中
    5. 在knowldegBase\mkdocs目录下创建一个mkdocs.yml
      1. 设置源文件目录docs_dir: 'docs'
    6. mkdocs gh-deploy 发布网页到github上. 打开https://我的账号.github.io/mkdocs/即可

    TIPS: 1. 我都是在wsl下操作的 2. docs_dir目录下的所有md文件都会被转换, 即使没有在yml里没有给出链接. 3. 可以利用pip3 install mkdocs-monorepo-plugin插件实现多个yml的嵌套 

        nav:
      - 首页: index.md
      - 兴趣:
        - Installation: getting-started.md
      - 工作:
        - Changing the colors: setup/changing-the-colors.md
      - 办公:
        - Abbreviations: reference/abbreviations.md
        - mkdocs: mkdocs语法.md
      - Insiders:
        - Sponsorship: insiders/index.md
        - v1: '!include ./v1/mkdocs.yml'
        - v2: '!include ./v2/mkdocs.yml'
    实际的目录如下: 
    .
├── docs
├── material-mkdocs-github.txt
├── mkdocs.yml
├── site
├── v1
└── v2
    其中v1和v2目录下含有yml文件和doc文件夹.
