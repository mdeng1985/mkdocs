    ROS空间¶

    创建ROS空间¶

    在sd卡上直接操作¶

    最直接的办法就是直接在树莓派的内存卡上操作。

    mkdir ~/catkin_ws_pi
cd ~/catkin_ws_pi
mkdir src
cd ~/catkin_ws_pi
catkin_make

    挂载NAS空间¶

    我这里直接运行的话，提示Permission denied: '/home/ubuntu/catkin_ws/build'。不太好解决。

    如果是树莓派，则 先切换到su账户，（否则在后面catkin_make的时候会提示Permission denied: '/home/ubuntu/catkin_ws/build') 然后需要： 

    echo "source /opt/ros/noetic/setup.bash" >> ~/.bashrc
source ~/.bashrc
    如果不输入上面的话，则运行catkin_make的时候找不到该命令。关于catkin_make权限问题，参见：http://wiki.ros.org/catkin/commands/catkin_make--

    sudo mount -o username="admin",password="Hxxd1..密码x" //192.168.0.139/NAS /mnt/nas2
ln -s 
/mnt/nas2/sync/mdeng/WORK/MyCode/linux_wsl/CODE/catkin_ws   ~/catkin_ws
cd ~/catkin_ws/
catkin_make #初始化工作空间

    WSL下操作¶

    cd /mnt/d/Dropbox/mdeng/WORK/MyCode/linux_wsl/CODE
mkdir -p catkin_ws/src
#创建软链接 源地址  目的地址
ln -s /mnt/d/Dropbox/mdeng/WORK/MyCode/linux_wsl/CODE/catkin_ws  ~/catkin_ws
cd ~/catkin_ws/
catkin_make #初始化工作空间
source devel/setup.bash
echo $ROS_PACKAGE_PATH #查看是否包含有工作目录

    ROS空间文件结构¶

    catkin_ws空间说明¶

    运行 

    cd ~/catkin_ws
sudo apt install tree
tree
    会看到通过tree命令可以看到catkin工作空间的结构,它包括了src、build、devel三个路径，在有些编译选项下也可能包括其他。但这三个文件夹是catkin编译系统默认的。它们的具体作用如下：

    • src/: ROS的catkin软件包（源代码包）
    • build/: catkin（CMake）的缓存信息和中间文件
    • devel/: 生成的目标文件（包括头文件，动态链接库，静态链接库，可执行文件等）、环境变量
