    0-ROS系统安装¶

    纯正的ubuntu下安装¶

    see: Ubuntu install of ROS Melodic

    添加sources.list¶

    (添加清华的) 

    sudo sh -c '. /etc/lsb-release && echo "deb http://mirrors.tuna.tsinghua.edu.cn/ros/ubuntu/ `lsb_release -cs` main" > /etc/apt/sources.list.d/ros-latest.list'

    添加key¶

    (可能需要关闭ss的代理) 

        sudo apt-key adv --keyserver 'hkp://keyserver.ubuntu.com:80' --recv-key C1CF6E31E6BADE8868B172B4F42ED6FBAB17C654

    sudo apt-get update¶

    如果提示The following signatures couldn't be verified because the public key is not available: NO_PUBKEY F42ED6FBAB17C654, 则需要运行:

    sudo apt-key adv --keyserver keyserver.ubuntu.com --recv-keys   F42ED6FBAB17C654

    安装ROS¶

      sudo apt-get install ros-melodic-desktop-full
  sudo apt-get install ros-melodic-desktop

    注意:

    • ubuntu 18版本对应的ros版本为melodic
    • 之所以安装一个full安装一个普通的，是有可能会丢失文件，参考
    • 如果为ubuntu 20.03，则安装sudo apt install ros-noetic-desktop-full

    rosdep 初始化¶

    (可能需要关闭ss的代理) 

    sudo rosdep init
rosdep update

    注意:

    • init有sudo，update没有sudo
    • rosdep update的作用就是在账户的根目录（home/ubuntu）下建立.ros文件夹
    • 如果提示rosdep: command not found
      • 如果是noetic的版本，则会提示找不到命令，这时候需要安装sudo apt install python3-rosdep 而不是python3-rosdep2 参考
      • 如果是melodic, 则sudo apt install python-rosdep2
    • 如果提示website may be down,则需要 
      sudo nano /etc/hosts
      加进去199.232.28.133 raw.githubusercontent.com （前者为后者的IP,ip查询地址）
    • 如果rosdep update提示超时等信息，则需要挂代理，首先看我的电脑-设置里的代理的端口是啥，我的是10801，则export https_proxy=http://127.0.0.1:10801 http_proxy=http://127.0.0.1:10801 all_proxy=socks5://127.0.0.1:10801，然后再执行rosdep update即可。

    配置环境变量¶

    echo "source /opt/ros/melodic/setup.bash" >> ~/.bashrc
source ~/.bashrc
    注意

    1. 如果是noetic版本，则将melodic换成noetic版本。
    2. echo "source /opt/ros/melodic/setup.bash" >> ~/.bashrc的含义是将source /opt/ros/melodic/setup.bash写入到bashrc文件里

    可以通过export | grep ROS来查看ROS环境变量是否正确 

    declare -x ROSLISP_PACKAGE_DIRECTORIES="/home/ubuntu/catkin_ws_pi/devel/share/common-lisp"
declare -x ROS_DISTRO="noetic"
declare -x ROS_ETC_DIR="/opt/ros/noetic/etc/ros"
declare -x ROS_MASTER_URI="http://localhost:11311"
declare -x ROS_PACKAGE_PATH="/home/mdeng1985/catkin_ws/src:/home/mdeng1985/CworkSpace/catkin_ws/src:/home/ubuntu/catkin_ws_pi/src:/opt/ros/noetic/share"
declare -x ROS_PYTHON_VERSION="3"
declare -x ROS_ROOT="/opt/ros/noetic/share/ros"
declare -x ROS_VERSION="1"

    构建工厂依赖¶

    sudo apt-get install python-rosinstall python-rosinstall-generator python-wstool build-essential
    如果是noetic，则sudo apt-get install python3-rosinstall python3-rosinstall-generator python3-wstool build-essential

    测试，运行¶

    roscore
    如果提示started core service [/rosout]则表明安装成功。

    测试可视化，如果是wsl2环境，则参考[[6-linux远程桌面连接#^1d69bc]]
