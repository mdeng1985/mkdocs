    turtlesim 实践¶

    关于package路径和node路径¶

    $rospack find turtlesim
/opt/ros/noetic/share/turtlesim
$ roscd turtlesim
cmake  images  msg  package.xml  srv
$ cd /opt/ros/noetic/lib/turtlesim
$ls 
draw_square  mimic  turtlesim_node  turtle_teleop_key

    因此turtlesim位于/opt/ros/noetic/share/turtlesim, 哪里又package.xml，则这个路径就是包的路径。

    但是实际上，节点的可执行文件的路径是位于/opt/ros/noetic/lib/turtlesim，可以看出有4个可执行文件:

    • draw_square
    • mimic
    • turtlesim_node
    • turtle_teleop_key

    运行节点的方式¶

    运行turtlesim节点的两种方式:

    1. rosrun turtlesim turtlesim_node
    2. cd /opt/ros/noetic/lib/turtlesim
./turtlesim_node

    3. 一个节点为一个可执行文件， rosnode list查看正在运行的节点列表

    标准语法如下： rosrun package-name executable-name

    TIPS:

    • 节点名称并不一定和可执行文件的名称相同。节点名称是可以自己定义的： rosrun package-name executable-name_name:=node-name
    • 运行turtlesim_node需要图形界面，因此如果是wsl下运行，则DISPLAY=:0 rosrun turtlesim turtlesim_node (see: [[6-linux远程桌面连接#^1d69bc|xming图形化显示]])
    • turtlesim_node是可执行文件的名称，它对应的节点名为turtlesim

    查看话题、消息¶

    TIPS:

    • 消息(msg)存放在话题(topci)里，
    • 分享消息时候，发布消息到对应话题
    • 接收信息时，订阅话题

    使用rqt_graph命令可以查看不同节点的消息传递关系

    ![[Pasted image 20201026205126.png]]

    • 椭圆形的是节点, 该节点即turtlesim (为什么不是turtlesim_node)
    • 矩形的是话题
      • turtle1/pose
      • turtle1/color_sensor
      • turtle1/cmd_vel

    查看当前活跃的话题： 

    ubuntu@ubuntu:~$ rostopic list
/rosout
/rosout_agg
/statistics
/turtle1/cmd_vel
/turtle1/color_sensor
/turtle1/pose

    打印话题里的消息rostopic echo topic-name

    比如: rostopic echo /turtle1/pose

    • 获取更多关于话题的信息rostopic info topic-name
    ubuntu@ubuntu:~$ rostopic info turtle1/pose
Type: turtlesim/Pose # 消息类型

Publishers: 
 * /turtlesim (http://ubuntu:40937/)

Subscribers: None

    查看消息类型的详细情况rosmsg show message-type-name 

    ubuntu@ubuntu:~$ rosmsg show turtlesim/Pose
float32 x
float32 y
float32 theta
float32 linear_velocity
float32 angular_velocity

    TIPS:turtlesim/Pose表示的turtlesim包里的Pose类型。

    理解ROS节点、消息和话题¶

    启动turtlesim在三个不同的终端中，分别执行以下三个指令：

    roscore

rosrun turtlesim turtlesim_node

rosrun turtlesim turtle_teleop_key

    如果切换到第三个终端（执行命令为turtle_teleop_key），然后按上、下、左、或右键，海龟就会响应你的命令开始移动了，并在屏幕上留下运动轨迹。

    总结¶

    1. ROS系统自带的功能包位于/opt/ros/noetic/share目录下（rospack find)
    2. ROS系统自带的功能包的功能包的可执行文件位于/opt/ros/noetic/lib目录下
    3. 运行节点的命令：rosrun package-name executable-name （注意最后一个是可执行文件的名称，不是节点名称）
