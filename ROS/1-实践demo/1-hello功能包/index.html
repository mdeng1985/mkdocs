    hello 功能包¶

    创建包¶

    catkin_create_pkg beginner_tutorial std_msgs rospy roscpp

    这样新建了一个包beginner_tutorial, 运行rosls beginner_tutorial得到 CMakeLists.txt include package.xml src

    rosls显示的就是src/beginner_tutorial路径下的目录

    这样在CMakeList.txt里就有： 

    ## Find catkin macros and libraries
## if COMPONENTS list like find_package(catkin REQUIRED COMPONENTS xyz)
## is used, also find other catkin packages
find_package(catkin REQUIRED COMPONENTS
    roscpp
    rospy
    std_msgs
)
    在package.xml里就有了 
    <build_depend>roscpp</build_depend>
<build_depend>rospy</build_depend>
<build_depend>std_msgs</build_depend>

    其中：

    • CMakeList.txt里的依赖项是自己编译包需要用的；package.xml里的依赖项是发布给别人用的。

    关于catkin_create_pkg的使用说明，可以参考[[3-ROS常用命令#catkin_create_pkg 创建包]]

    编辑源文件¶

    编辑工作空间/src/beginner_tutorial/src/hello.cpp文件 (可以利用vscode remote wsl进行编辑)

        #include<ros/ros.h>

    int main(int argc, char ** argv)
    {
    // Initialize the ROS system .
    ros::init(argc,argv,"hello_ros");//hello_ros为节点默认名
    // Establ ish this program as a ROS node .
    ros::NodeHandle nh;//该命令将本程序注册为ROS节点管理器的节点
    // Send some output as a log message .
    ROS_INFO_STREAM("Hello, ROS!");//生成一条消息
    }

    -ros::init(argc,argv,"hello_ros");里的hello_ros为默认节点的名称。

    编辑CMakeList.txt¶

    声明可执行文件 

    add_executable(executable-name source-files)
target_link_libraries(executable-name ${catkin_LIBRARIES})

    因此在CMakeList的末尾加上 

        add_executable(hello src/hello.cpp)
    target_link_libraries(hello ${catkin_LIBRARIES})

    • hello.cpp路径是相对于CMakeList.txt所在目录的路径

    编译工作区¶

    cd ~/catkin_ws
catkin_make

    _CATKIN_SETUP_DIR

    更新环境变量¶

    Sourcing setup.bash

    source devel/setup.bash

    这样ROS才能找到创建的包和可执行文件

    执行hello程序¶

    rosrun beginner_tutorial hello

    hello为beginner_tutorial包下的可执行文件，对应的节点的hello_ros。

    注意：需要事先运行roscore

    可执行文件的路径： catkin_ws_pi/devel/lib路径下。

    总结¶

    • 如果是已经完成编写了，则按照如下的步骤：
    cd ~/catkin_ws
catkin_make
source devel/setup.bash
rosrun beginner_tutorial hello
    • 可执行文件的路径： catkin_ws/devel/lib路径下。
