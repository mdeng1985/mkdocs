    WSL下的安装¶

    Installation in Linux 该网站还给出了一些可选设置，我没有去试。

    其中测试的时候需要设置环境变量 

        set OPENCV_TEST_DATA_PATH environment variable to <path to opencv_extra/testdata>.
    execute tests from build directory.

    我设置环境变量的方法为：sudo nano ~/.bashrc,在末尾加上 

        export OPENCV_TEST_DATA_PATH=~/CworkSpace/opencv/opencv-3.4.10/build/opencv_extra/testdata
    然后source ~/.bashrc

    弄完以后确认下是否添加了环境变量echo $OPENCV_TEST_DATA_PATH

    如果不添加环境变量，则会有的3个地方失败 

    [  FAILED  ] 3 tests, listed below:
[  FAILED  ] Core_InputOutput.filestorage_base64_basic_read_XML
[  FAILED  ] Core_InputOutput.filestorage_base64_basic_read_YAML
[  FAILED  ] Core_InputOutput.filestorage_base64_basic_read_JSON

    其他环境变量: export OPENCV_TEST_DATA_PATH=~/CworkSpace/opencv/opencv-3.4.10/build/opencv_extra/testdata export OPENCV_SAMPLES_DATA_PATH=~/CworkSpace/opencv/opencv-3.4.10/samples/data

    开发环境搭建¶

    创建opencv工作目录¶

    主要参考cmake工作目录创建方法， see TAGS_CMAKE_PRACTICE

    • 创建一个的demo项目 
          mkdir opencv_ws
    mkdir src build bin lib
    cd src
    mkdir demo1
    • vscode wsl打开目录opencv_ws

    • 编辑的demo1的源文件DisplayImage.cpp，相应的CMakeLists.txt 内容如下 

          cmake_minimum_required(VERSION 2.8)
    project( DisplayImage )
    SET(CMAKE_BUILD_TYPE "Debug")
    SET(CMAKE_CXX_FLAGS_DEBUG "$ENV{CXXFLAGS} -O0 -Wall -g2 -ggdb")
    SET(CMAKE_CXX_FLAGS_RELEASE "$ENV{CXXFLAGS} -O3 -Wall")

    find_package( OpenCV REQUIRED )
    SET(EXECUTABLE_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/../../bin) 

    include_directories( ${OpenCV_INCLUDE_DIRS} )
    add_executable( DisplayImage DisplayImage.cpp )
    target_link_libraries( DisplayImage ${OpenCV_LIBS} )

    • 把lena.jpg放到opencv_ws/bin目录下

    • bash下进行编译运行 

          cd build
    cmake ../src/demo1
    make
      这样在opencv_ws/bin下生存了可执行文件DisplayImage

    • 设置显示环境变量

      • 开启MobtaXtem，查看display的ip，比如192.168.1.106:0.0
      • wsl远程终端bash下设置DISPLAY环境变量export DISPLAY=192.168.1.106:0.0

    • 运行程序，bash下回到opencv_ws目录, 然后运行 

          ../bin/DisplayImage bin/lena.jpg
      注意：执行程序时, lena.jpg应为相对于bash的当前目录的路径，而不是相对于DisplayImage的路径，因此后面的 bin不能少。

    VSCODE-WSL调试环境搭建¶

    • 首先存储vscode workspace, 路径为opencv_ws/opencv.code-workspace,内容默认即可 
          {
    "folders": [
        {
            "path": "."
        }
    ],
    "settings": {}
}
    • ctrl+shift+p调出c_cpp_properties.json,内容为 
          {
        "configurations": [
            {
                "name": "Linux",
                "includePath": [
                    "${workspaceFolder}/**"
                ],
                "defines": [],
                "compilerPath": "/usr/bin/gcc",
                "cStandard": "gnu11",
                "cppStandard": "gnu++14",
                "intelliSenseMode": "clang-x64"
            }
        ],
        "version": 4
    }
    • 首先点击vscode左侧的debug按钮,然后点create a launch.json (有两个选项一个是opencv_ws另外一个是workspace,我选的是workspace),内容如下: 
              {
        // Use IntelliSense to learn about possible attributes.
        // Hover to view descriptions of existing attributes.
        // For more information, visit: https://go.microsoft.com/fwlink/?linkid=830387
        "version": "0.2.0",
        "configurations": [
            {
                "name": "(gdb) Launch",
                "type": "cppdbg",
                "request": "launch",
                "program": "${workspaceFolder}/bin/DisplayImage",
                "args": [
                    "bin/lena.jpg"
                ],
                "stopAtEntry": false,
                "cwd": "${workspaceFolder}",
                "environment": [],
                "externalConsole": false,
                "MIMode": "gdb",
                "setupCommands": [
                    {
                        "description": "Enable pretty-printing for gdb",
                        "text": "-enable-pretty-printing",
                        "ignoreFailures": true
                    }
                ]
            }
        ]
    }
      注意:
    • program为可执行文件路径,

    • args为传递的参数, bin/lena.jpg即为argv[1]

    • 然后生成可执行文件, bash切换到 opencv_ws目录下,调试即可.
