    关于欧式变换¶

    \[ \newcommand \bm \mathbf \]

    $$ \begin{align} \bm{b} =\bm{R}_A^B\bm{a} + \bm{t}^B \end{align}

    $$ 其中：

    • \(\bm{a}\)为某一点在A坐标系下的坐标
    • \(\bm{b}\)为某一点在B坐标系下的坐标
    • \(\bm{R}_A^B\) 为A坐标系到B坐标系下的姿态转换矩阵
    • \(\bm{t}^B\)为B坐标系原点到A坐标系原点的矢量在B坐标系下的投影

    上式的物理意义据说：A坐标系先旋转，然后平移\(-\bm{t}_B\)

    上式可以写成欧氏变换的矩阵形式 $$ \begin{align}\label{atob} \bm{b}=\bm{T}\bm{a} \end{align} $$ 其中 $$ \begin{align} \bm{T} = \left[\begin{array}{cc} \bm{R}A^B & \bm{t}^B \ \bm{0} & 1 \end{array} \right] \end{align} $$

    实践¶

    关于**视觉SLAM十四讲**里的例3.6.2中的几点说明

    1. \(q_1\)对应的是R1到W世界坐标系的姿态四元数
    2. \(t_1\)为R1坐标系原点到W世界坐标系原点的矢量在R1坐标系下的投影

    关于EIGEN库的几点说明

    1. Quaterniond q1(0.35, 0.2, 0.3, 0.1)第一个数为标量，后3个为矢量
    2. Eigen中的矩阵默认是列优先
    3. 假设q1为A到B的姿态四元数，则q1.toRotationMatrix()得到的B到A的姿态矩阵。

    Eigen构建欧氏变换的方法¶

    比如得到W到R1的欧氏变换

    1. 首先给出R1到W的姿态四元数\(q_1\)，得到一个欧氏变换矩阵Isometry3d T1w(q1)​
    2. 然后平移即可T1w.pretranslate(t1)

    另外一种方法：就是直接利用旋转+平移去计算。

    两种方法的代码如下：(路径D:\Dropbox\mdeng\WORK\MyCode\linux_wsl\CODE\cmake_ws\src\slambook2\ch3\src可以在vs2019下远程编译)

    #include <iostream>
#include <vector>
#include <algorithm>
#include <Eigen/Core>
#include <Eigen/Geometry>

using namespace std;
using namespace Eigen;

int main(int argc, char **argv){
    // 坐标变换的例子
    Vector3d t1(0.3, 0.1, 0.1), t2(-0.1, 0.5, 0.3); 
    Vector3d p1(0.5, 0, 0.2);
    Quaterniond q1(0.35, 0.2, 0.3, 0.1), q2(-0.5, 0.4, -0.1, 0.2);
    q1.normalize();
    q2.normalize();

    //方法1
    Matrix3d Cr1tw = q1.toRotationMatrix().inverse();
    Vector3d tmp3 = p1 - t1;
    Vector3d xw = Cr1tw * tmp3;
    Matrix3d Cwtr2 = q2.toRotationMatrix();
    Vector3d pR2 = Cwtr2 *(xw) + t2;
    cout << pR2 << endl;
    //slambook2中的方法
    Isometry3d T1w(q1), T2w(q2);
    T1w.pretranslate(t1);
    T2w.pretranslate(t2);
    Vector3d ptmp = T1w.inverse() * p1;
    Vector3d p2 = T2w * T1w.inverse() * p1;
    cout << p2 << endl;    
}
