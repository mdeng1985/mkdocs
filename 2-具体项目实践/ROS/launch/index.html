    launch 文件学习¶

    ros¶

    launch文件用于将多个运行的节点打包起来一并运行。

    roslaunch命令介绍¶

    roslaunch运行启动文件的命令如下： 

        roslaunch package-name launch-file-name

    roslaunch在启动节点之前会判断roscore是否运行，如果没有则自动启动roscore

    roslaunch的命令行参数说明：

    http://wiki.ros.org/cn/roslaunch/Commandline%20Tools

    • roslaunch -p 1234 package filename.launch 在1234上端口运行
    • --wait 等到监测到roscore再开始运行launch
    • --local 只运行local nodes，不运行远程机器上的nodes
    • --screen Force all node output to screen. Useful for node debugging.
    • -v输出详细信息
    • -dump-params Print parameters in launch file in YAML format.

    其他 

        cd `roslaunch-logs`
    可以快速切换到log的目录

    更多的参数见[[0-Tutorial Using roslaunch to start Gazebo, world files and URDF models]]

    创建launch文件¶

    launch文件的位置¶

    launch文件一般放在package的根目录或者launch目录下

    比如roscd pack-name切换到package的根目录下查看

    基本元素¶

         <launch>
         ...
    </launch>

    启动节点 

        <node
    pkg=”package-name” # 功能包的名称
    type=”executable-name” # 可执行文件的名称
    name=”node-name” # 别名？
    />

    demo示例 

        1 <launch>
    2 <node
    3 pkg="turtlesim "
    4 type="turtlesim_node"
    5 name="turtlesim "
    6 respawn="true "
    7 />
    8 <node
    9 pkg="turtlesim "
    10 type="turtle_teleop_key"
    11 name="teleop_key"
    12 required="true "
    13 launch −prefix="xterm −e"
    14 />
    15 <node
    16 pkg="agitr "
    17 type="subpose"
    18 name="pose_subscriber "
    19 output="screen "
    20 />
    21 </launch>

    一些设置 - respawn="true"表示如果节点停止了，则会重新启动该节点 - required="true" 表示该节点为必要节点，一旦该节点终止，则roslaunch会终止其他节点。required和respawn不能同时配置，否则会出错。 - launch −prefix="xterm −e" 表示开启新的终端运行该节点

    启动文件中的其他元素¶

    包含其他文件¶

        <include file="path-to-launch-file">
    或者 xml <include file="$(find package-name)/launch-file-name">

    ### 启动参数

    args，有点像可执行程序中的局部变量， launch中有两种编写方法

        <arg name=”arg-name” default=”arg-value”/>
    <arg name=”arg-name” value=”arg-value”/>

    arg标签官方文档

    知识点： - arg变量只在启动文件中才有意义 - arg 变量后面是default的时候，在命令行中可以对该参数进行重新赋值，value则不能重新赋值。 - arg变量的赋值有两种方式，一种是放在launch文件中，如上所示；另外一种是在命令行下，比如roslaunch package-name launch-file-name arg-name:=arg-value - $(arg arg-name)得到arg-name参数的值。roslaunch在遇到这个的时候，会自动翻译成参数值。

    group¶

    group的作用有两点

    1. 组可以把若干个节点放入同一个命名空间内

        <group ns=”namespace”/>
    …
    </group>
    3. 组可以有条件地使能或禁用一个节点。

        <group if=”0 or 1”/>
    </group>

    注意: - if后面必须为1或者0 - if也可以用unless代替，只不过执行条件相反

    gazebo_ros里的launch实例解析¶

    mud_world.launch¶

        <launch>
  <!-- We resume the logic in empty_world.launch, changing only the name of the world to be launched -->
  <include file="$(find gazebo_ros)/launch/empty_world.launch">
    <arg name="world_name" value="worlds/mud.world"/> <!-- Note: the world_name is with respect to GAZEBO_RESOURCE_PATH environmental variable -->
    <arg name="paused" value="false"/>
    <arg name="use_sim_time" value="true"/>
    <arg name="gui" value="true"/>
    <arg name="recording" value="false"/>
    <arg name="debug" value="false"/>
  </include>
</launch>

    知识点：

    • include 标签之间的参数是属于被包含文件

    由于传递变量都在include以内，所以这些变量的值都会传递给empty_world.launch里的对应变量。以world_name变量为例进行说明，该launch文件的含义表明是将值worlds/mu.world传递给empty_world.launch中的world_name变量

    下面看empty_world.launch 

    <?xml version="1.0"?>
    <launch>

      <!-- these are the arguments you can pass this launch file, for example paused:=true -->
      <arg name="paused" default="false"/>
      <arg name="use_sim_time" default="true"/>
      <arg name="extra_gazebo_args" default=""/>
      <arg name="gui" default="true"/>
      <arg name="recording" default="false"/>
      <!-- Note that 'headless' is currently non-functional.  See gazebo_ros_pkgs issue #491 (-r arg does not disable
           rendering, but instead enables recording). The arg definition has been left here to prevent breaking downstream
           launch files, but it does nothing. -->
      <arg name="headless" default="false"/>
      <arg name="debug" default="false"/>
      <arg name="physics" default="ode"/>
      <arg name="verbose" default="false"/>
      <arg name="output" default="screen"/>
      <arg name="world_name" default="worlds/empty.world"/> <!-- Note: the world_name is with respect to GAZEBO_RESOURCE_PATH environmental variable -->
      <arg name="respawn_gazebo" default="false"/>
      <arg name="use_clock_frequency" default="false"/>
      <arg name="pub_clock_frequency" default="100"/>
      <arg name="enable_ros_network" default="true" />

      <!-- set use_sim_time flag -->
      <param name="/use_sim_time" value="$(arg use_sim_time)"/>

      <!-- set command arguments -->
      <arg unless="$(arg paused)" name="command_arg1" value=""/>
      <arg     if="$(arg paused)" name="command_arg1" value="-u"/>
      <arg unless="$(arg recording)" name="command_arg2" value=""/>
      <arg     if="$(arg recording)" name="command_arg2" value="-r"/>
      <arg unless="$(arg verbose)" name="command_arg3" value=""/>
      <arg     if="$(arg verbose)" name="command_arg3" value="--verbose"/>
      <arg unless="$(arg debug)" name="script_type" value="gzserver"/>
      <arg     if="$(arg debug)" name="script_type" value="debug"/>

      <!-- start gazebo server-->
      <group if="$(arg use_clock_frequency)">
        <param name="gazebo/pub_clock_frequency" value="$(arg pub_clock_frequency)" />
      </group>
      <group>
        <param name="gazebo/enable_ros_network" value="$(arg enable_ros_network)" />
      </group>
      <node name="gazebo" pkg="gazebo_ros" type="$(arg script_type)" respawn="$(arg respawn_gazebo)" output="$(arg output)"
            args="$(arg command_arg1) $(arg command_arg2) $(arg command_arg3) -e $(arg physics) $(arg extra_gazebo_args) $(arg world_name)" />

      <!-- start gazebo client -->
      <group if="$(arg gui)">
        <node name="gazebo_gui" pkg="gazebo_ros" type="gzclient" respawn="false" output="$(arg output)" args="$(arg command_arg3)"/>
      </group>

    </launch>

    该launch内容的解释可以分为如下几个部分： 1. arg 2. param 3. group 4. node

    先看下面的部分 

        <arg unless="$(arg paused)" name="command_arg1" value=""/>
    <arg     if="$(arg paused)" name="command_arg1" value="-u"/>
    含义为： 如果变量paused的值为1，则command_arg1的值为-u，否则为空

    • 类似的，可以得到command_arg2的值为-r或者空
    • command_arg3的值为--verbose或者空
    • script_type的值为gzserver或者debug

    假设command_arg1~3的值为-u,-r,--verbose, scrpt_type的值为debug, 则下面语句的含义为 

        <node name="gazebo" pkg="gazebo_ros" type="$(arg script_type)" respawn="$(arg respawn_gazebo)" output="$(arg output)"
            args="$(arg command_arg1) $(arg command_arg2) $(arg command_arg3) -e $(arg physics) $(arg extra_gazebo_args) $(arg world_name)" />


        <node name="gazebo" pkg="gazebo_ros" type="debug" respawn="$(arg respawn_gazebo)" output="$(arg output)"
            args="-u -r --verbose -e $(arg physics) $(arg extra_gazebo_args) $(arg world_name)" />
    args="-u -r --verbose -e $(arg physics) $(arg extra_gazebo_args) $(arg world_name)"对应的是命令行参数。

    可参考：http://wiki.ros.org/cn/roslaunch/Commandline%20Tools
