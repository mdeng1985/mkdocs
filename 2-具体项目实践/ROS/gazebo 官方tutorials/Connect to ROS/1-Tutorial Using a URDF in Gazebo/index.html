    Tutorial: Using a URDF in Gazebo¶

    Unified Robotic Description Format (URDF)为XML格式，用于在ROS中描述机器人。

    Gazebo可以自动将URDF转换成SDF

    背景¶

    URDF格式 - 可以描述质点运动和动力学特性， - 不能描述机器人在世界里的pose - 不能描述oint loops (parallel linkages), and it lacks friction and other properties - 不能描述非机器人，比如光线、高程图等。

    为了克服URDF格式的缺点，创建了Simulation Description Format (SDF)格式用于Gazebo。 SDF格式描述了元素涵盖世界和机器人，格式为XML.

    RRBot 示例¶

    一般的步骤是首先在Rviz下测试URDF模型,然后再在Gazebo中对机器人模型进行配置.

    参考:ROS URDF Tutorials.

    下载 RRBot¶

    cd ~/catkin_ws/src/
git clone https://github.com/ros-simulation/gazebo_ros_demos.git
cd ..
catkin_make

    其他参考： ROS Overview Tutorials.

    Rviz里和Gazebo里查看模型¶

    在Rviz下和Gazbo下查看的命令如下:

    roslaunch rrbot_description rrbot_rviz.launch
roslaunch rrbot_gazebo rrbot_world.launch

    TIPS: - 可能需要安装sudo apt install ros-melodic-joint-state-publisher-gui - 如果看不到，则试着强制关掉其他进程killall roscore 再试试 (这个很有效) - gazebo_ros_control 教程里给出了如何利用Rviz来监视机器人利用/joint_states发布的状态。在本例中，Rviz里的RRbot接受虚拟的joint_states_publisher节点发布的/joint_states

    launch文件解析¶

    rrbot_rviz.launch¶

    <launch>
  <param name="robot_description"
    command="$(find xacro)/xacro --inorder '$(find rrbot_description)/urdf/rrbot.xacro'" />

  <!-- send fake joint values -->
  <node name="joint_state_publisher" pkg="joint_state_publisher" type="joint_state_publisher">
    <param name="use_gui" value="TRUE"/>
  </node>

  <!-- Combine joint values -->
  <node name="robot_state_publisher" pkg="robot_state_publisher" type="robot_state_publisher"/>

  <!-- Show in Rviz   -->
  <node name="rviz" pkg="rviz" type="rviz" args="-d $(find rrbot_description)/launch/rrbot.rviz"/>

</launch>

    主要功能见最后一个运行节点,运行rviz包的rviz节点,调用的是rrbot.rviz文件, 其中该文件中的参数robot_description里给出了机器人的模型,该参数是传递给rrbot.rviz

    rrbot_world.launch¶

    这个是Gazebo里的启动文件, 内容如下: 

    <launch>

  <!-- these are the arguments you can pass this launch file, for example paused:=true -->
  <arg name="paused" default="false"/>

  <arg name="gui" default="true"/>
  <arg name="headless" default="false"/>
  <arg name="debug" default="false"/>

  <!-- We resume the logic in empty_world.launch, changing only the name of the world to be launched -->
  <include file="$(find gazebo_ros)/launch/empty_world.launch">
    <arg name="world_name" value="$(find rrbot_gazebo)/worlds/rrbot.world"/>
    <arg name="debug" value="$(arg debug)" />
    <arg name="gui" value="$(arg gui)" />
    <arg name="paused" value="$(arg paused)"/>
    <arg name="use_sim_time" value="$(arg use_sim_time)"/>
    <arg name="headless" value="$(arg headless)"/>
  </include>

  <!-- Load the URDF into the ROS Parameter Server -->
  <param name="robot_description"
    command="$(find xacro)/xacro --inorder '$(find rrbot_description)/urdf/rrbot.xacro'" />

  <!-- Run a python script to the send a service call to gazebo_ros to spawn a URDF robot -->
  <node name="urdf_spawner" pkg="gazebo_ros" type="spawn_model" respawn="false" output="screen"
    args="-urdf -model rrbot -param robot_description"/>

  <!-- ros_control rrbot launch file -->
  <!--include file="$(find rrbot_control)/launch/rrbot_control.launch" /-->

</launch>
    其中 - 世界模型为: rrbot.world - 运行spwan_model节点生成机器人, 该机器人模型由robot_description给出.

    xacro文件中link标签的详细解析¶

    ^efb6c3

    首先需要熟悉URDF link 元素。示例如下：

      <!-- Base Link -->
  <link name="link1">
    <collision>
      <origin xyz="0 0 ${height1/2}" rpy="0 0 0"/>
      <geometry>
        <box size="${width} ${width} ${height1}"/>
      </geometry>
    </collision>

    <visual>
      <origin xyz="0 0 ${height1/2}" rpy="0 0 0"/>
      <geometry>
        <box size="${width} ${width} ${height1}"/>
      </geometry>
      <material name="orange"/>
    </visual>

    <inertial>
      <origin xyz="0 0 1" rpy="0 0 0"/>
      <mass value="1"/>
      <inertia
        ixx="1.0" ixy="0.0" ixz="0.0"
        iyy="1.0" iyz="0.0"
        izz="1.0"/>
    </inertial>
  </link>

    几点说明 - <collision> 和 <visual> 元素的内容基本是一样的, 这是因为使得模型既在Gazebo中能显示又能在Rviz中显示. 对于Gazebo来说, 如果不指定<collision>,则默认为不可见的. - <visual> 还指定了材质, <material name="orange"/>, orange用于rviz的定义见materials.xacro; orange用于Gazebo的定义见rrbot.gazebo, 详细解读见[[1-Tutorial Using a URDF in Gazebo#^813610|material 材质定义]] - Gazebo里的单位为m和kg REP 103: Standard Units of measure and Coordinate Conventions - <inertial>给出了质量特性, 这个是用于Gazebo(我的理解)物理引擎URDF link element , 一些设置可以在Gazebo-View菜单里选择''Wireframe'' 和 ''Center of Mass''查看是否选择正确。一般可以利用Solidworks等CAD对惯量进行近似;;

    material 材质定义¶

    ^813610

    利用如下的命令指定颜色，

    <material name="orange"/>

    orange颜色在单独的文件里定义，比如materials.xacro:

      <material name="orange">
    <color rgba="${255/255} ${108/255} ${10/255} 1.0"/>
  </material>

    这种方法在Gazebo里不适用，在Gazebo里需要采用如下的标签：

      <gazebo reference="link1">
    <material>Gazebo/Orange</material>
  </gazebo>

    RRBot示例中调用 rrbot.gazebo文件，在该文件中给出了相应的和Gazebo相关的标签，比如 <link> and <material> .

    • Gazebo的默认可用材质见 gazebo/media/materials/scripts/gazebo.material.
    • 如果要自己创建材质， 见：
    • The <material> SDF documentation
    • OGRE materials documentation

    关节Joints详细解读¶

    ^56ef76

    下面给出joint1的定义 

        <joint name="joint1" type="continuous">
        <parent link="link1"/>
        <child link="link2"/>
        <origin xyz="0 ${width} ${height1 - axel_offset}" rpy="0 0 0"/>
        <axis xyz="0 1 0"/>
        <dynamics damping="0.7"/>
    </joint>

    首先需要熟悉 URDF joint documentation. 对于Gazebo来说，需要特别注意如下的几点：

    • <origin>, <parent> 和 <child> 是必须的
    • 省去了<calibration> 和 <safety_controller>
    • 在 <dynamics> 标签里，gazebo4只用了 damping 属性， Gazebo5 及以上版本还用了 friction 属性
    • 在 <limit> 的属性是可选项
    • dynamics damping=0.7 N\*m\*s/rad

    rrbot.gazebo解读¶

    <?xml version="1.0"?>
<robot>

  <!-- ros_control plugin -->
  <gazebo>
    <plugin name="gazebo_ros_control" filename="libgazebo_ros_control.so">
      <robotNamespace>/rrbot</robotNamespace>
      <robotSimType>gazebo_ros_control/DefaultRobotHWSim</robotSimType>
    </plugin>
  </gazebo>

  <!-- Link1 -->
  <gazebo reference="link1">
    <material>Gazebo/Orange</material>
  </gazebo>

  <!-- Link2 -->
  <gazebo reference="link2">
    <mu1>0.2</mu1>
    <mu2>0.2</mu2>
    <material>Gazebo/Black</material>
  </gazebo>

  <!-- Link3 -->
  <gazebo reference="link3">
    <mu1>0.2</mu1>
    <mu2>0.2</mu2>
    <material>Gazebo/Orange</material>
  </gazebo>

  <!-- camera_link -->
  <gazebo reference="camera_link">
    <mu1>0.2</mu1>
    <mu2>0.2</mu2>
    <material>Gazebo/Red</material>
  </gazebo>

  <!-- hokuyo -->
  <gazebo reference="hokuyo_link">
    <sensor type="gpu_ray" name="head_hokuyo_sensor">
      <pose>0 0 0 0 0 0</pose>
      <visualize>false</visualize>
      <update_rate>40</update_rate>
      <ray>
        <scan>
          <horizontal>
            <samples>720</samples>
            <resolution>1</resolution>
            <min_angle>-1.570796</min_angle>
            <max_angle>1.570796</max_angle>
          </horizontal>
        </scan>
        <range>
          <min>0.10</min>
          <max>30.0</max>
          <resolution>0.01</resolution>
        </range>
        <noise>
          <type>gaussian</type>
          <!-- Noise parameters based on published spec for Hokuyo laser
               achieving "+-30mm" accuracy at range < 10m.  A mean of 0.0m and
               stddev of 0.01m will put 99.7% of samples within 0.03m of the true
               reading. -->
          <mean>0.0</mean>
          <stddev>0.01</stddev>

    验证模型是否转换成SDF¶

    运行下面的命令查看URDF是否转换成SDF格式。

      # gazebo2 and below
  gzsdf print MODEL.urdf
  # gazebo3 and above
  gz sdf -p MODEL.urdf

    Note: in Gazebo version 1.9 and greater, some of the debug info has been moved to a log file you can view with:

      cat ~/.gazebo/gzsdf.log

    在 Gazebo里查看URDF模型¶

    Viewing the RRBot in Gazebo was already covered at the beginning of this tutorial. For your own custom robot, we assume its URDF lives in a ROS package named MYROBOT_description in the subfolder /urdf. The method to open a URDF from that location into Gazebo using ROS was covered in the previous tutorial, Using roslaunch Files to Spawn Models. If you have not completed that tutorial, do so now.

    From that tutorial you should have two ROS packages for your custom robot: MYROBOT_description and MYROBOT_gazebo. To view your robot and test it in Gazebo, you should be able to now run something like:

    roslaunch MYROBOT_gazebo MYROBOT.launch

    This should launch both the Gazebo server and GUI client with your robot automatically launched spawned inside.

    调整¶

    参考：SDF user guide

    If your robot model behaves unexpectedly within Gazebo, it is likely because your URDF needs further tuning to accurately represent its physics in Gazebo. See the SDF user guide for more info on various properties available in Gazebo, which are also available in the URDF via the <gazebo> tag.

    共享模型¶

    If you have a common robot that other's might want to use in Gazebo, you are encouraged to add your URDF to the Gazebo Model Database. It is an online server that Gazebo connects to to pull down models from the internet. Its Mercurial repository is located on GitHub. See Gazebo Model Database documentation for how to submit a pull request to have your robot added to the database.
