    发布和订阅消息¶

    消息说明¶

    每一个消息类型都有一个相对应 C++头文件。格式如下： 

    #include <package_name/type_name.h>
    比如：#include <geometry_msgs/Twist.h> 在使用的时候利用：：来对包名和类型名进行分开。

    说明Twist消息是属于geometry_msgs包的。

    发布消息¶

    源程序¶

    注意：如果一个节点发布多个话题的消息，需要创建多个publisher对象。

    因此发布消息程序的几个步骤如下：

    1. 声明要发布的消息类型：#include<geometry_msgs/Twist.h>
    2. 创建并初始化节点： 
      ros::init(argc,argv,"publlish_velocity");
    ros::NodeHandle nh;
    3. publlish_velocity为节点名
    4. nh为节点对象
    5. 在节点对象上创建一个发布者对象 
      ros::Publisher pub = nh.advertise<geometry_msgs::Twist>( "turtle1/cmd_vel", 1000);
    6. pub为发布者对象
    7. geometry_msgs::Twist为消息类型
    8. turtle1/cmd_vel为发布的话题

    9. 1000 为消息序列的大小，设置为一个大数 标准形式： 

      ros::Publisher pub = node_handle.advertise<message_type>(
topic_name, queue_size);

    10. 创建并填充消息对象 

          geometry_msgs::Twist msg;
        msg.linear.x = double(rand()) / double(RAND_MAX);
msg.angular.z = 2 * double(rand()) / double(RAND_MAX) - 1;
      利用rosmsg show geometry_msgs/Twist得到 
      geometry_msgs/Vector3 linear
  float64 x
  float64 y
  float64 z
geometry_msgs/Vector3 angular
  float64 x
  float64 y
  float64 z
      因此可以设置msg.linear.x, msg.angular.z

    11. 发布消息 

                        pub.publish(msg);

    完整的源文件如下： 在beginner_tutorial目录下的src目录下新建pubvel.cpp文件，内容如下 

    //This program publishes randomly-generated velocity messages for turtlesim
#include<ros/ros.h>
#include<geometry_msgs/Twist.h>//For geometry_msgs::Twsit
#include<stdlib.h>//for rand and RAND_MAX

int main(int argc, char **argv)
{
    ros::init(argc,argv,"publlish_velocity");
    ros::NodeHandle nh;

    ros::Publisher pub = nh.advertise<geometry_msgs::Twist>( "turtle1/cmd_vel", 1000);
    srand(time(0));
    ros::Rate loop_rate(2);
    while (ros::ok())
    {
        geometry_msgs::Twist msg;
        msg.linear.x = double(rand()) / double(RAND_MAX);
        msg.angular.z = 2 * double(rand()) / double(RAND_MAX) - 1;

        pub.publish(msg);

        ROS_INFO_STREAM("Sending random velocity command : "
                        << " linear=" << msg.linear.x
                        << " angular=" << msg.angular.z);

        // Wait untilit's time for another iteration .
        loop_rate.sleep();
    }

}

    编译执行流程¶

    由于pubvel.cpp里需要includegeometry_msgs包，因此在beginner_tutorial所在目录的 package.xml里加上

    <build_depend>geometry_msgs</build_depend>
<build_export_depend>geometry_msgs</build_export_depend>
<exec_depend>geometry_msgs</exec_depend>

    然后CMakeLists.txt里加上 

        find_package(catkin REQUIRED COMPONENTS
    roscpp
    rospy
    std_msgs
    geometry_msgs
    )
    add_executable(pubvel src/pubvel.cpp)
    target_link_libraries(pubvel  ${catkin_LIBRARIES})
    **Step **编译工作区

        cd ~/catkin_ws
    catkin_make

    Step Sourcing setup.bash

        source devel/setup.bash

    **Step**执行pubvel程序

    rosrun beginner_tutorial pubvel

    pubvel为beginner_tutorial包下的可执行文件

    **Step**开启三个终端，分别运行 - roscore - rosrun turtlesim turtlesim_node - rosrun beginner_tutorial pubvel 就可以看到小海龟在动了。

    订阅者消息¶

    程序解读¶

    订阅消息流程如下： 1. 首先要声明的消息类型#include<turtlesim/Pose.h> 2. 创建并初始化节点 

    ros::init(argc,argv,"subscribe_to_pose");
        ros::NodeHandle nh;
    3. 创建订阅者对象 
    ros::Subscriber sub = nh.subscribe("turtle1/pose",1000, &poseMessageReceive);
    - turtle1/pose为要订阅的话题 - poseMessageReceive为有消息到达时调用的函数，需要自己编写。 
    ros::Subscriber sub = node_handle.subscribe
(topic_name,queue_size, pointer_to_callback_function);
    - ros::spin();表示给ros控制权，和下面的语句等同 
        while(ros::ok( ))
{
ros::spinOnce();
}

    其中回调函数的poseMessageReceive写法如下： 

    void poseMessageReceive( const turtlesim::Pose& msg)
    {
        ROS_INFO_STREAM( std::setprecision ( 2 ) << std::fixed
    << " position =(" << msg . x << " , " << msg . y << " ) "
    << " *direction=" << msg . theta   );
    }
    - const turtlesim::Pose& msg表明msg的类型为turtlesim::Pose

    源程序如下： （beginner_tutorial的src目录下创建subpose.cpp）

    // This program subscribes to turtle1/pose and shows its
// messages on the screen .
#include <ros/ros.h>
#include<turtlesim/Pose.h>
#include<iomanip>// for std::setprecision and std::fixed

// A callback function . Executed each time a new pose
// message arrives .
void poseMessageReceive( const turtlesim::Pose& msg)
    {
        ROS_INFO_STREAM( std::setprecision ( 2 ) << std::fixed
    << " position =(" << msg . x << " , " << msg . y << " ) "
    << " *direction=" << msg . theta   );
    }

int main(int argc, char** argv){
        ros::init(argc,argv,"subscribe_to_pose");
        ros::NodeHandle nh;
        ros::Subscriber sub = nh.subscribe("turtle1/pose",1000, &poseMessageReceive);
        ros::spin();
    }
