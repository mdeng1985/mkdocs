    初级¶

    WSL2下运行¶

    主要是确保小海龟和可视化能够正常运行。

    rosrun rviz rviz

    见linux_wsl2安装.md里的笔记安装MobaXterm 用xming也可以，见[[6-linux远程桌面连接#^1d69bc]]

    消息(msg)和服务(srv)介绍¶

    • msg位于package的msg目录下
    • srv文件位于package的srv目录下

    msg样例

      Header header
  string child_frame_id
  geometry_msgs/PoseWithCovariance pose
  geometry_msgs/TwistWithCovariance twist

    srv文件分为请求和响应两部分，由---分隔

    int64 A
int64 B
---
int64 Sum

    创建一个msg¶

    • 首先进入到packag目录下，创建msg目录，创建msg文件
    $ cd ~/catkin_ws/src/beginner_tutorials
$ mkdir msg
$ echo "int64 num" > msg/Num.msg
    • 添加package.xml依赖以，生成C++,PYTHON等语言
    <build_depend>message_generation</build_depend>
<exec_depend>message_runtime</exec_depend>
    • 编辑CMakeLists.txt,添加三部分
    find_package(catkin REQUIRED COMPONENTS roscpp rospy std_msgs message_generation)

    add_message_files(
  FILES
  Num.msg
)

    generate_messages()

    • 显示msg详细类型rosmsg show beginner_tutorials/Num

    详解服务¶

    • 服务和话题不一样，服务是客户端先发送请求，服务器再根据请求返回响应；
    • 属于单次通信

    • 列出当前活动的服务清单rosservice list

    • 然后可以看到有/turtlesim1/spawn

    • 查看/turtlesim1/spawn服务的类型

    rosservice type /turtlesim1/spawn

    得到turtlesim/Spawn

    • 然后查看服务请求和相应格式
    mdeng1985@LAPTOP-HUFVUD3P:~/catkin_ws$ rossrv info turtlesim/Spawn
float32 x
float32 y
float32 theta
string name
---
string name
    • 然后输入rosservice call /turtlesim1/spawn 2.0 2.0 2.0 'test'就可以再生一个小海龟，名字为test. (调用/turtlesim1/spawn服务，参数为.....以再生一个小海龟)

    编写简单的服务器和客户端 (C++)¶

    Step1 定义srv文件¶

    • 进入功能包，创建srv目录
    $ roscd beginner_tutorials
$ mkdir srv
    • 新建AddTwoInts.srv文件，内容如下
    int64 a
int64 b
---
int64 sum
    • CMakeLists.txt做出如下修改
    # Do not just add this line to your CMakeLists.txt, modify the existing line
find_package(catkin REQUIRED COMPONENTS
  roscpp
  rospy
  std_msgs
  message_generation)

    add_service_files(
  FILES
  AddTwoInts.srv
)

    generate_messages(
  DEPENDENCIES
  std_msgs
)

    std_msgs为消息文件依赖的msg

    • rossrv show beginner_tutorials/AddTwoInts

    Step2 创建服务端节点¶

    新建add_two_ints_server.cpp内容如下

    #include "ros/ros.h"
#include "beginner_tutorial/AddTwoInts.h"

bool add(beginner_tutorial::AddTwoInts::Request  &req,
         beginner_tutorial::AddTwoInts::Response &res)
{
  res.sum = req.a + req.b;
  ROS_INFO("request: x=%ld, y=%ld", (long int)req.a, (long int)req.b);
  ROS_INFO("sending back response: [%ld]", (long int)res.sum);
  return true;
}

int main(int argc, char **argv)
{
  ros::init(argc, argv, "add_two_ints_server");
  ros::NodeHandle n;

  ros::ServiceServer service = n.advertiseService("add_two_ints", add);
  ROS_INFO("Ready to add two ints.");
  ros::spin();

  return 0;
}

    然后在CmakeLists.txt里添加

    # 服务端
add_executable(add_two_ints_server src/add_two_ints_server.cpp)
target_link_libraries(add_two_ints_server ${catkin_LIBRARIES})
add_dependencies(add_two_ints_server beginner_tutorial_gencpp)

    Step3 创建客户端节点¶

    客户端名称为add_two_ints_client.cpp

    #include "ros/ros.h"
#include "beginner_tutorial/AddTwoInts.h"
#include <cstdlib>

int main(int argc, char **argv)
{
  ros::init(argc, argv, "add_two_ints_client");
  if (argc != 3)
  {
    ROS_INFO("usage: add_two_ints_client X Y");
    return 1;
  }

  ros::NodeHandle n;
  ros::ServiceClient client = n.serviceClient<beginner_tutorial::AddTwoInts>("add_two_ints");
  beginner_tutorial::AddTwoInts srv;
  srv.request.a = atoll(argv[1]);
  srv.request.b = atoll(argv[2]);
  if (client.call(srv))
  {
    ROS_INFO("Sum: %ld", (long int)srv.response.sum);
  }
  else
  {
    ROS_ERROR("Failed to call service add_two_ints");
    return 1;
  }

  return 0;
}

    CMakeLists.txt内容添加：

    add_executable(add_two_ints_client src/add_two_ints_client.cpp)
target_link_libraries(add_two_ints_client ${catkin_LIBRARIES})
add_dependencies(add_two_ints_client beginner_tutorial_gencpp)

    Step4编译和运行服务¶

    进入到catkin_ws目录

    catkin_make
source devel/setup.bash

    开启一个终端，开启服务rosrun beginner_tutorial add_two_ints_server

    打开另外一个终端，运行客户端rosrun beginner_tutorial add_two_ints_client 4 5

    启动文件¶

    在beginner_tutorial文件夹下新建example.launch，内容如下：

    <launch>
<node
pkg="turtlesim"
type="turtlesim_node"
name="turtlesim"
respawn="true"
/>
<node
pkg="turtlesim"
type="turtle_teleop_key"
name="teleop_key"
required="true"
/>
<node
pkg="beginner_tutorial"
type="subpose"
name="pose_subscriber"
output="screen"
/>
</launch>

    然后在~/catkin_ws目录下运行bash: roslaunch beginner_tutorial example.launch

    标准格式：

    roslaunch package-name launch-file-name
    • 在启动任意节点之前， roslaunch 首先会判断 roscore 是否正在运行； 如果没有， 则自动 启动 roscore
    • 每一个启动文件都应该和一个特定的功能包关联起来,最简单的方法是把启动文件直接存储在功能包的根目录中
    • respawn 属性为真，这样当节点停止的时候， roslaunch 会重新启动该节点。
    • 由于 respawn 和 required 二者的作用是相互矛盾的，因此，如果对一个节点同时配置了这两种属性，roslaunch会报错
    • xml文件里可以设置命名空间，以分别运行独立的launch： ns=”namespace”
    • 如果输出详细信息，则roslaunch -v

    node的格式

    <node
pkg=”package-name”
type=”executable-name”
name=”node-name”
/>
