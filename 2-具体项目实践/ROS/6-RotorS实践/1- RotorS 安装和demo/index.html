    RotorS 安装和demo¶

    mav #ros #rotorS¶

    安装和demo运行¶

    • 安装一些必要的包：

      • 安装glog 
        sudo apt install libgoogle-glog-dev
      • 安装gazebo_ros_pkgs工具包 
        sudo apt-get install ros-melodic-gazebo-ros-pkgs ros-melodic-gazebo-ros-control
        安装完的位置位于/opt/ros/melodic/share/gazebo_ros
      • 安装octomap 参考 
        sudo apt-get install ros-melodic-octomap
      • 安装octomap_ros 
        cd ~/catkin_ws/src
git clone https://github.com/OctoMap/octomap_ros
cd ..
catkin_make
      • 安装octomap-msgs 
        sudo apt-get install ros-melodic-octomap-msgs
        -的

    • 下载和安装rotorS，把如下的4个文件夹下载至catkin_ws/src/rotorS目录下 

      catkin_simple  
ethzasl_msf  
glog_catkin  
mav_comm  
rotors_simulator  
rotors_simulator_demos
      catkin_simple下载地址 后面的下载地址

    • 安装 

        cd ~/catkin_ws
  catkin_make
  source devel/setup

    • 将~/.ignition/fuel/config.yaml中的api.ignitionfuel.org替换为fuel.ignitionrobotics.org
    • 运行demo 
          mdeng1985@LAPTOP-T9AM7QKA:~/catkin_ws$  roslaunch rotors_gazebo mav_hovering_example.launch
      注意：
    • 在wsl下，只有采用vcxsrv方式才能显示gazebo，采用mobaxterm和xming都无法正常显示。安装过程见[[6-linux远程桌面连接#^dc0292]]
    • 再开启一个窗口运行gazebo的时候需要输入 
       export GAZEBO_IP=127.0.0.1 #也很关键
 export DISPLAY=$(cat /etc/resolv.conf | grep nameserver | awk '{print $2}'):0
 export LIBGL_ALWAYS_INDIRECT=0 # 这个很关键
 export DISPLAY=127.0.0.1:0.0
      奇怪的是每次需要都需要重新运行这4个命令，然后再运行roslanuch才能显示；放到bashrc里是不行的。

    关于launch文件的详细解读见[[2-mav_hovering_example.launch详解]]

    总结¶

    下次运行demo的操作如下： 

     export GAZEBO_IP=127.0.0.1 #也很关键
 export DISPLAY=$(cat /etc/resolv.conf | grep nameserver | awk '{print $2}'):0
 export LIBGL_ALWAYS_INDIRECT=0 # 这个很关键
 export DISPLAY=127.0.0.1:0.0
 cd ~/catkin_ws
 source devel/setup.bash
 roslaunch rotors_gazebo mav_hovering_example.launch
