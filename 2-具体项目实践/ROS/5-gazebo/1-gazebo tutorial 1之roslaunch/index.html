    Tutorial: Using roslaunch to start Gazebo, world files and URDF models¶

    Tutorial: Using roslaunch to start Gazebo, world files and URDF models 给出了roslaunch 启动gazebo_ros的一些参数变量

    首先给出demo world的启动方式，终端输入任意一个下面的命令 

        roslaunch gazebo_ros mud_world.launch
    roslaunch gazebo_ros shapes_world.launch
    roslaunch gazebo_ros rubble_world.launch

    TIPS: - 运行roslaunch会自动运行roscore - roscd gazebo_ros/launch && ls -la可以看到所有的launch清单

    [[launch#roslaunch]]和[[launch]]文件剖析¶

    首先roscd gazebo_ros , 然后sudo nano lanuch/mud_world.launch打开launch文件, 内容如下： 

        <launch>
      <!-- We resume the logic in empty_world.launch, changing only the name of the world to be launched -->
      <include file="$(find gazebo_ros)/launch/empty_world.launch">
        <arg name="world_name" value="worlds/mud.world"/> <!-- Note: the world_name is with respect to GAZEBO_RESOURCE_PATH environmental variable -->
        <arg name="paused" value="false"/>
        <arg name="use_sim_time" value="true"/>
        <arg name="gui" value="true"/>
        <arg name="recording" value="false"/>
        <arg name="debug" value="false"/>
      </include>
    </launch>

    该luanch文件的解析可以参见[[launch]]

    world_name变量的值未worlds/mud.world，关于world文件的位置，可以参考[[0-gazebo里的world]]

    world内容如下 

        <sdf version="1.4">
    <world name="default">
      <include>
        <uri>model://sun</uri>
      </include>
      <include>
        <uri>model://ground_plane</uri>
      </include>
      <include>
        <uri>model://double_pendulum_with_base</uri>
        <name>pendulum_thick_mud</name>
        <pose>-2.0 0 0 0 0 0</pose>
      </include>
      ...
    </world>
  </sdf>
