    linux下cmake工作空间目录结构¶

    模仿catkin_ws创建如下的文件目录 （tree -n -L 2命令可以显示） 

        .
    ├── bin
    │   ├── hello
    │   └── libtest
    ├── build
    │   └── CMakeFiles
    ├── cmake.code-workspace
    ├── libs
    │   └── libStudent.so
    └── src
        ├── helloworld
        ├── student
        ├── studentSharedLib
        └── testLibrary
    其中： - src文件夹存放项目源文件cpp和项目自身的头文件h，由不同的子项目组成 - libs文件夹存放生成的libs - build文件夹存放编译临时文件 - bin文件存放生成的可执行文件
    - cmake.code-workspace 为vscode的工作空间

    注1: src下的子目录下还可以分为include和src目录用于对应该子项目的头文件和源文件

    注2：我的cmake_ws目录是用vscode-remote wsl下建立的，步骤如下： - 创建软链接ln -s /mnt/d/Dropbox/mdeng/WORK/MyCode/linux_wsl/CODE /home/mdeng1985/CworkSpace - mkdir cmake_ws - 保存vscode的workspace:cmake.code-workspace

    **注3**下次使用的时候，在wsl-remote里打开code-workspace就可以了。

    具体项目实践¶

    student项目¶

    • 以student项目为例进行说明， 模仿catkin_ws创建如下的文件目录 （tree命令可以显示）
    .
  ├── bin
  ├── build
  └── src
      └── student
          ├── CMakeLists.txt
          ├── include
          │   └── Student.h
          └── src
              ├── Student.cpp
              └── mainapp.cpp

    创建CMakeLists.txt: 在student目录下的CMakeLists.txt内容如下： 

        cmake_minimum_required(VERSION 3.5.1)
    project(studentTest)
    include_directories(include)
    # set(SOURCES src/mainapp.cpp src/Student.cpp)
    file(GLOB SOURCES "src/*.cpp")
    SET(EXECUTABLE_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/../../bin) 
    add_executable(testStudent  ${SOURCES})

    其中 - ${PROJECT_SOURCE_DIR}路径为CMakeLists.txt所在目录 - 其他目录是相对于CMakeLists.txt所在目录的相对路径 -

    • 在vscode remote wsl的terminal： cd build
    • 然后cmake ../src/student/
    • make会生成在bin文件夹下生成teststudent的可执行文件
    • make clean清空build目录下的文件(或者直接到上级目录，删除build目录rm -rf ../build/*)
    • ../bin/testStudent即可运行可执行文件。

    vscode下项目的调试¶

    CMAKE编译¶

    CmakeLists.txt的内容如下： 

        cmake_minimum_required(VERSION 3.5.1)
    project(Hello)
    SET(CMAKE_BUILD_TYPE "Debug")
    SET(CMAKE_CXX_FLAGS_DEBUG "$ENV{CXXFLAGS} -O0 -Wall -g2 -ggdb")
    SET(CMAKE_CXX_FLAGS_RELEASE "$ENV{CXXFLAGS} -O3 -Wall")

    SET(EXECUTABLE_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/../../bin) 
    add_executable(hello helloworld.cpp)

    -cd build: cmake ../src/helloworld

    • make会在cmake_ws/bin文件夹下生成hello可执行文件
    • 然后在helloworld.cpp里设置一个断点

    VSCODE端的设置¶

    首先安装c/c++插件

    • 点vscode左侧的调试按钮，选择open a folder打开cmake_ws然后会在该目录下创建一个.vscode的隐藏目录，在下面创建一个launch.json用于程序调试用。
    • 编辑launch.json里设置可执行文件的名字：**"program": "${workspaceFolder}/bin/hello"即可调试。

    这里的cmake workspace为d:\Dropbox\mdeng\WORK\MyCode\linux_wsl\CODE\cmake_ws\

    调试的时候需要首先在bash下切换到cmake_ws目录
