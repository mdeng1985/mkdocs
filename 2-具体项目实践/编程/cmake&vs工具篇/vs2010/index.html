    生成库¶

    演练：创建和使用静态库 (C++)

    库按编译位数可以生成64位和32位；按照模式可以生成debug模式和release模式，因此总共可以生成4种库。

    首先按照说明新建solution:StaticLibrary，该solution下有个project位MathfunctionLib。默认生成的库是放在solution下的文件夹内。可以自己设置：

    项目的属性里-configuration properties --,genral - output directory
    我设置成$(SolutionDir)$(Platform)\$(Configuration)\，其中平台对应64和32，configuration对应relase还是debug。

    注意在进行上面设置的时候，先将configuration设置成all configuration。这样对于debug和relase都是这样了。

    最后生成库了

    |-- Win32
|   |-- Debug
|   |   `-- MathFuncsLib.lib
|   `-- Release
|       `-- MathFuncsLib.lib
`-- x64
    |-- Debug
    |   `-- MathFuncsLib.lib
    `-- Release
        `-- MathFuncsLib.lib

    使用库¶

    几点说明和注意

    • Build 下的Configuration Manager设置的是要运行的模式和平台版本；项目右键属性里也有配置和平台信息，那个是对某一配置和平台下的进行相关设置。

    • Build 下的Configuration Manager下上面是solution的运行模式，下面是project的运行模式，统一设置成一样。比如说solution是release的x64，把下面的也设置成relase和x64. 要确保build下面的对号是选上的。参考下图

    • 关于包含路径：右键具体的项目，在c/c++下面general目录下Additional includes设置包含目录 （release /debug，win32/x64下都要设置，设置4次）

    • 关于库路径和名字：右键具体的项目，在linker-general-additonallibrary directories下设置库目录，然后在linker-input-additional dependence里设置库文件。两点注意事项:

      • 库目录和configuration和pltaform对应，比如configuration选的是realse，platform选的是win32，那么调用的库也需要是realease,win32下生成的库。
      • 注意库文件名的后缀.lib不能省掉，否则编译会提示LINK : fatal error LNK1104: cannot open file 'MathFuncsLib.obj'
    • 上面添加包含路径和库等可以通过添加引用的方式来实现：选中项目-然后项目-reference-add new reference (这样好像需要引用的和被引用位于同一个solution下。)

    vs添加设置所有项目的包含目录、库文件¶

    对于有些标准的库和包含文件，需要设置成所有项目都包含。添加方式如下：

    view --property manager-- debug| Win32- microsoft.CPP.win32.user

    同样的，在x64平台也需要添加 view --property manager-- debug| x64- microsoft.CPP.win64.user

    包含目录好像debug和release可以公用，库目录debug和release还需要单独设置。

    cl编译并运行一个简单的程序¶

    cl为vs对应的编译器，编写如下的prog.cpp

            #include <iostream>

        int main()
        {
        std::cout << "Enter two numbers:" << std::endl;
            return 0;
        }

    然后调用cl进行编译，在编译之前需要设置环境变量

    • Bin (对应cl文件的目录)
            D:\Program Files (x86)\Microsoft Visual Studio 12.0\VC\bin\amd64
    • lib
            d:\Program Files (x86)\Microsoft Visual Studio 12.0\VC\lib\amd64\
        C:\Program Files (x86)\Windows Kits\8.1\Lib\winv6.3\um\x64
    • include
            D:\Program Files (x86)\Microsoft Visual Studio 12.0\VC\include
    • PATH
            %bin%
        %lib%
        %include%

    然后在cmd下进入到cpp所对应的文件夹

        cl prog1.cpp

    就会生成prog1.exe,然后cmd下运行prog1就会显示程序运行的输出。

    如果出现下面的错误：

    fatal error LNK1112: module machine type 'X86' conflicts with target machine type 'x64'

    表明环境变量设置有问题，cl是x86的，而lib设置的是x64的。需要将两者统一起来，要么都设置成x64，要么设置成x86.

    FAQ(实践过程中出现的问题和解决方案记录)¶

    • 安装vs2015以后找不到系统自带的包含目录和库文件

    视图-其他窗口-属性管理器-debug|Win32-Win32.user-Vc++目录添加如下四个目录

    C:\Program Files (x86)\Windows Kits\10\Include\10.0.10150.0\ucrt
C:\Program Files (x86)\Windows Phone Kits\8.1\Include
C:\Program Files (x86)\Microsoft Visual Studio 14.0\VC\include
C:\Program Files (x86)\Microsoft Visual Studio 14.0\VC\atlmfc\include

    然后在库目录下添加如下的目录：

    C:\Program Files (x86)\Windows Kits\10\Lib\10.0.10150.0\ucrt\x86

    同理在Debug|X64的的属性管理器也进行类似的设置，只不过把库目录改成

    C:\Program Files (x86)\Windows Kits\10\Lib\10.0.10150.0\ucrt\x64
