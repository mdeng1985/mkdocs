    vscode下的rtklib编译¶

    [toc]

    首先参考cmake实践篇进行cmake_ws工作空间的创建

    rtklib的编译和调试¶

    rtklib库编译¶

    在cmake_ws/src/下新建rtklib文件夹，把rtklib的源文件和头文件放到该文件夹里：

    .
├── CMakeLists.txt
├── include
└── src

    CMakeLists.txt的内容如下：

    ​ 

    cmake_minimum_required(VERSION 3.5.1)
project(rtklib)

# set(SOURCES src/mainapp.cpp src/Student.cpp)
SET(CMAKE_BUILD_TYPE "Debug")
SET(CMAKE_CXX_FLAGS_DEBUG "$ENV{CXXFLAGS} -O0 -Wall -g2 -ggdb")
SET(CMAKE_CXX_FLAGS_RELEASE "$ENV{CXXFLAGS} -O3 -Wall")

include_directories(include)
file(GLOB SOURCES "src/*.c" "src/rcv/*.c")
SET(LIBRARY_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/../../libs) 
LINK_LIBRARIES(-lpthread -lm)
add_library(rtklib SHARED  ${SOURCES})

    然后到build目录下

    • cmake ../src/rtklib/
    • make
    • 这样就在cmake_ws/libs下编译成功了librtklib.so

    rtklib测试¶

    创建另外的项目rtklibTest, 其cMakeLists.txt内容如下

    cmake_minimum_required(VERSION 3.5.1)
project(rtkLibTest)

SET(CMAKE_BUILD_TYPE "Debug")
SET(CMAKE_CXX_FLAGS_DEBUG "$ENV{CXXFLAGS} -O0 -Wall -g2 -ggdb")
SET(CMAKE_CXX_FLAGS_RELEASE "$ENV{CXXFLAGS} -O3 -Wall")




set (PROJECT_LINK_LIBS librtklib.so) 
link_directories(../../libs)

include_directories(include)

# set(SOURCES src/mainapp.cpp src/Student.cpp)

file(GLOB SOURCES "src/*.c")

SET(EXECUTABLE_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/../../bin) 
add_executable(rtkLibTest  ${SOURCES})
target_link_libraries(rtkLibTest ${PROJECT_LINK_LIBS} )

    1. 然后到build目录下

    2. cmake ../src/rtklibTest/

    3. make

    4. 这样就在cmake_ws/bins下编译成功了rtklibTest

    5. 然后点调试按钮，设置launch.json的"program": "${workspaceFolder}/bin/rtklibTest"

    6. crtl+shift+P然后c/c++ edit configuration，把includepath添加进去 比如"${workspaceFolder}/src/rtklibTest/include" (不设置这个调试的时候无法跳转)
    7. 调试即可

    其他¶

    1. 函数定义无法跳转的问题：crtl+shift+P然后c/c++ edit configuration，把includepath添加进去 比如"${workspaceFolder}/src/testRTKlib/include"
    2. 无法智能提示，file-preference-setting搜prevent，然后把 Snippets prevent quick suggestinos取消选中
    3. CMAKE无法设置断点的问题，提示"module containing this breakpoint has not yet loaded or the breakpoint address could not be obtained"，这时候修改cMakeLists.txt文件的内容如下:SET(CMAKE_CXX_FLAGS_DEBUG "$ENV{CXXFLAGS} -Wall -ggdb3 -O0 -std=c99") 参考资料https://github.com/Microsoft/vscode-cpptools/issues/416
