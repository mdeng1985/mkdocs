    设置调试模式¶

    SET(CMAKE_BUILD_TYPE "Debug")
SET(CMAKE_CXX_FLAGS_DEBUG "$ENV{CXXFLAGS} -O0 -Wall -g2 -ggdb")
SET(CMAKE_CXX_FLAGS_RELEASE "$ENV{CXXFLAGS} -O3 -Wall")

    打印信息¶

    message(STATUS "PROJECT_SOURCE_DIR is" ${PROJECT_SOURCE_DIR})

    设置相关路径¶

    SET(EXECUTABLE_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/../../bin) 
SET(LIBRARY_OUTPUT_PATH ${PROJECT_SOURCE_DIR}/../../../libs)
include_directories(include)

    设置库文件的路径还有一种方法：

    link_directories(../../libs)

    CMAKE里的变量¶

    DIR变量¶

    • CMAKE_SOURCE_DIR
      CMAKE source tree的根目录，我的理解就是源程序的根目录。 [see](https://cmake.org/cmake/help/v3.0/variable/CMAKE_SOURCE_DIR.html)
    • CMAKE_BINARY_DIR
      `CMAKE_BINARY_DIR`为CMake BUILD tree的根目录。比如说：我们在`CMAKE_SOURCE_DIR/build`目录下build，那么`CMAKE_BINARY_DIR`就是对应`CMAKE_SOURCE_DIR/bin`。也可以人为设置成其他目录，比如说：


            set(CMAKE_BINARY_DIR ${CMAKE_SOURCE_DIR}/bin) 将其设置为源程序根目录下的bin文件夹
    see

    PATH变量¶

    • lib文件的输出路径LIBRARY_OUTPUT_PATH

    变量的使用方法¶

    • set方法

    设置B变量对应的目录为A变量，B变量的格式需要加上${}, A不需要。而且B还能嵌套使用，参考下面的例子。其中第二个例子来自于suitesparse的Cmakelists.txt

    set(CMAKE_BINARY_DIR ${CMAKE_SOURCE_DIR}/bin)
SET(LIBRARY_OUTPUT_PATH ${${PROJECT_NAME}_BINARY_DIR}/lib CACHE PATH "Output directory for libraries" )
    第二行中的CACHE PATH,表示当用户如果命令行进行指定，那优先会去读用户的命令行.see

    如果是下面的命令好像就是强制设置

            SET(CMAKE_INSTALL_PREFIX "${SUITESPARSE_INSTALL_PREFIX}" CACHE INTERNAL "Prefix prepended to install directories" FORCE)

         set(SP_LIB_PREFIX "lib")   设置字符串lib为SP_LIB_PREFIX
