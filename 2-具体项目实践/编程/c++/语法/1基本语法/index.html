    变量的定义和声明¶

    • 定义是一种特殊的声明; 只能定义一次，声明可以多次声明

    • 声明指定了变量的名称和类型，而定义在这基础上还进一步分配了了内存，甚至还进行了初始化

    • 因此如果一个变量在声明的同时进行了初始化，则这个声明就是定义

    • 常常采用extern来指定声明，但是如果extern后面页进行了显式初始化，则为定义（说明显式初始化的优先级高于extern）

    extern in i; //声明
int j; //定义，因为默认是初始化了
extern int i = 3; //定义，因为进行了初始化
    参考资料：c++ primer 5th, section 2.2.2

    const限定符¶

    const限定符常用于限定如下几种类型：

    • 基本变量，如double，int等
    • 引用reference
    • 指针point

    const用于基本类型¶

    如果用于基本类型，则表明常量，一旦赋值以后就不允许改变了。有几点注意事项

    • const用于基本类型，则该常量为局部常量；因此如果多个文件用到了这个常量，则需要分别定义。
    • 如果为了避免在多个文件里多次定义const常量，则需要在前面加上关键词extern 
          extern const int buffSize = fcn();//cpp文件中 定义
    extern const int buffSize// .h头文件中

    const用于引用¶

    有两种情况，

    • 引用的为常量，这时候引用和常量都必须为const形式，如下 
          const int ci = 1024;
    const int &r1 = ci;//正确
    int &r2 =ci; // 错误，因为ci为const
    • 引用本身为const形式，但是引用的不为常量 
          double ci = 1024.0;
    const int &r1 = ci;//正确
      • r1和ci的类型可以不一样，一个是double，一个是int
      • 对于这种引用形式，表明不允许通过改变r1来改变ci的值，但是ci的值仍然可以改变了，一旦ci的值改变了，r1的值也会改变。

    const用于指针¶

    const用于指针也有两种情况：

    1. 指向常量的指针： 这时候格式为const int* p1=...，对于这种情形，不允许通过*p1改变它指向的变量的值。但是可以改变它的指向。 
          const int *p1 = &b; // b可以不为const型
    p1 = &bb1;//正确
    *p1 = 3;//错误
    2. 常量指针：这时候const在*后面，意味着该指针指向某个确定的变量（也即该指针存放的地址是固定的），但是该变量的值是可以改变的 
         int *const p2 = &b;//
   p2 = &bb1;//错误，p2为常指针，存放的地址是固定的
   *p2=3;//正确

    顶层const¶

    • 如果是顶层const, 则不允许改变；底层的可以改变；在引用中的const永远为底层const 
          int *const p1 = &i;//p1为常量指针，不允许改变p1存放的地址为顶层const
    const int ci =42//不允许改变ci的值，顶层const
    const int *p2 = &ci;//允许改变p2的值，这里的const为底层const
    const int &r2 = ci;// 在引用中的const永远为底层const
    • 执行将A赋值给B，如果A含有顶层const，则这个属性会被删除 {.cpp} const int ci =42//不允许改变ci的值，顶层const int cj = ci;//正确 int &r =ci;//错误，因为引用中的const永远为底层的，这个还没理解透彻。
