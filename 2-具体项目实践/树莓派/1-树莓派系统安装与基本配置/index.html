    linux系统安装¶

    linux系统： - 带界面的ubuntu mate (有界面) - ubuntu server

    开始尝试安装ubuntu mate但是无法启动，连接hdmi也没有显示。所以放弃。因此我这里安装ubuntu server

    安装步骤如下： 1. 下载ubuntu server for raspberry-pi (不要下载ubuntu mate) 我装的是20.04版本 2. 利用7zip解压成img（不要用winrar） 3. SD Memory Crad Formatter进行格式化，格式e:/BOOT。 4. 利用win32diskimager将镜像写入到sd卡里或Etcher（带有校验功能） 5. 打开sd卡里的config.txt文件，在末尾加上 

    Over_voltage =6
Arm_freq = 2000

    hdmi_mode = 82
    disable_overscan=1
    1. 将sd卡插到树莓派，树莓派通过micro hdmi与显示器连接。 2. 通电会启动linux命令行界面，默认的登录和密码均为ubuntu。可以修改密码修改，我自己的修改为hit1...1

    基本配置¶

    设置ubuntu用户权限¶

    将ubutnu用户的权限改成root一致。

    • 执行chmod 777 /etc/sudoers命令修改sudoers文件权限为可编辑权限。

    • 执行vi /etc/sudoers命令编辑sudoers文件。

    • 在root帐号下添加普通用户帐号: ubuntu ALL=(ALL) ALL

    • 编辑完成后，单击“Esc”，并输入:wq，保存文件并退出。

    • 执行chmod 440 /etc/sudoers命令恢复sudoers文件权限为只读权限。

    上网设置¶

    上网方式有两种

    • 树莓派连接网线，共享电脑的网络参考树莓派通过网线上网： 网络与internet设置---WLAN-网络与共享中心然后选WLAN设置。
    • 直接进行WiFi设置（更为便携）
      • 首选查看wifi card名:ls /sys/class/net,其中wlan0就是wifi card name
      • 修改网络配置文件 
        cd /etc/netplan/
cp 50-cloud-init.yaml 50-cloud-init.yaml.bak
sudo nano 50-cloud-init.yaml
        对wifi进行设置（wlan0) 
            network:
    version: 2
    ethernets:
        eth0:
            optional: true
            dhcp4: true
    # add wifi setup information here ...
    wifis:
        wlan0:
            optional: true
            access-points:
                "yuexin":
                    password: "wifi密码"
            dhcp4: true
        然后输入sudo netplan --debug generate看看是否由错误，注意编辑yaml的文件不要使用table键，否则调试会显示tabs are not allowed for indent:

    设置完以后可以输入sudo apt-get update看看是否连上网了。

    注意： 这里设置完以后，进到桌面里，WiFi里是看不到网络的。会影响x11vnc的使用。因此在安装完桌面环境以后需要将这个文件里的wlan部分给删了。利用桌面自带的wifi管理进行连接。

    参考资料： How to setup of Raspberry Pi 3 onboard WiFi for Ubuntu Server 18.04?

    How to Setup WiFi on Raspberry Pi 4 with Ubuntu 20.04 LTS 64-bit ARM Server

    更换软件源¶

    sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak
cd /etc/apt
sudo nano sources.list
    然后将里面ports.ubuntu.com都换成mirrors.aliyun.com，然后ctrl+o保存，ctrl+x退出

    更新源，然后重启： 

    sudo apt-get update
sudo apt-get upgrade
reboot

    安装桌面环境¶

    连上网以后就可以安装桌面环境了sudo apt-get install ubuntu-desktop （也可以按装其他桌面Which *buntu to pick?）

    然后reboot，就可以启动桌面环境了。

    安装中文输入法¶

    参考Ubuntu 安装中文输入法

    1. win + a 打开所有应用程序，找到 Language Support 并打开，在 kygboard input method system 中查看是否有 fcitx 。如果有，请设置为 fcitx ,如果没有，则手动安装：sudo apt-get install fcitx
    2. sudo apt install fcitx-googlepinyin
    3. 修复依赖关系：sudo apt-get install -f
    4. 重启win + a 打开所有应用，搜索fcitx configuration 并打开 打开input method 添加输入法即可。

    开机自动登录¶

    方式1（推荐）¶

    在桌面环境下，找到settings-users然后开启automatic login即可。

    方式2（不推荐）¶

    编辑/usr/share/lightdm/lightdm.conf.d/60-lightdm-gtk-greeter.conf在末尾行添加自己的账户，比如

    autologin-user = ubuntu

    然后再启动盘的config.txt里加上一行：hdmi_force_hotplug=1 (这个加上去，分辨率变差了。。。)

    **注意：**我后来安装的是ubuntu-desktop，没有60-lightdm-gtk-greeter.conf文件，这个没有实现。

    SSH登录¶

    树莓派连上网以后，在命令终端ifconfig -a查看ip，我的为192.168.1.107

    然后windows下putty登录（xshell登录更方便）即可。

    我的树莓派对应的mac地址如下（利用advance IP scanner扫描得到） 

        192.168.1.107   192.168.1.107               DC:A6:32:62:CF:53
        Ok  192.168.1.108   192.168.1.108           Raspberry Pi Foundation B8:27:EB:CC:5C:86        (树莓派3shu)
